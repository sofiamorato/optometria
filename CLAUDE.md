# Instrucciones para cualquier IA

## Antes de escribir código

Leer completamente:

- README.md
- ESTADO.md
- docs/PROYECTO.md
- docs/BASE_DATOS.md
- docs/UX.md
- docs/DECISIONES.md

Después continuar únicamente con la siguiente tarea pendiente.

---

# Objetivo

Desarrollar una aplicación web para una clínica de optometría.

El objetivo principal es conservar exactamente el flujo de trabajo actual de la doctora.

Nunca rediseñar el producto.

Nunca cambiar decisiones aprobadas.

---

# Usuario Final

Una optómetra con muy poca experiencia utilizando tecnología.

Todas las decisiones deben priorizar simplicidad.

---

# UX

Siempre:

- Letras grandes
- Botones grandes
- Muy pocos botones
- Una acción principal por pantalla
- Guardado automático
- Navegación paso a paso

Nunca:

- Dashboards
- Sidebars
- Formularios enormes
- Pantallas llenas de opciones

---

# Historia Clínica

Los PDFs ubicados en la carpeta "formatos" son la fuente oficial del proyecto.

Nunca modificar su estructura.

Si existe alguna duda sobre un campo, preguntar antes de implementarlo.

---

# Inteligencia Artificial

La IA solamente puede:

- Mejorar redacción
- Dictado
- Organizar texto
- Generar documentos usando datos existentes

La IA nunca puede:

- Inventar datos
- Diagnosticar
- Completar información clínica automáticamente

---

# Desarrollo

Trabajar siempre por etapas.

No implementar funcionalidades que no hayan sido solicitadas.

Mantener el código limpio y bien organizado.

Siempre explicar decisiones importantes antes de implementarlas.
